Essential Skills — ENG 6

Lecture 1 — Essential Skills

MATLAB fundamentals: variables, expressions, scripts, and your first functions.

Build fluency with these:
- Command Window and Interpreter Basics — enter expressions, read errors, use semicolons to suppress output
- Command History and Rapid Iteration — reuse and edit previous commands with the up-arrow to accelerate debugging
- Variable Creation, Assignment, and Workspace Inspection — create variables, use who/whos/clear to manage state
- Identifier Rules and Naming Best Practices — valid names, no leading digits or spaces, avoid shadowing built-ins
- Common Constants and Built-In Math Functions — pi, eps, sin, cos, sqrt, exp, abs, round

Expect to think harder here:
- Translating Problems Into MATLAB Workflows — break an engineering problem into steps: variables, computation, results; requires planning before writing a single line of code
- Numeric Types, Literals, and Expression Evaluation — operator precedence and type conversion; subtle precedence rules can cause silent errors
- Text Basics: String Scalars and Character Vectors — distinguish "..." from '...', concatenation, isstring/ischar; two similar types with different behaviors trip up most beginners
- Scripts, Comments, and Reproducible Code Style — clear structure, clc/clear, avoid relying on persistent workspace state; discipline required, not just syntax
- Simple Functions and Basic User Input — function files, calling from scripts, input, input validation; file structure and I/O together in one topic

Conceptually demanding — start early:
- Basic Number Representation Concepts — binary/finite-precision, rounding, dec2bin/bin2dec; counterintuitive: 0.1 + 0.2 ~= 0.3 in MATLAB surprises almost everyone

Lecture 2 — Essential Skills

Logical values, vectors, and array indexing fundamentals.

Build fluency with these:
- Logical Scalars and Boolean Variables — create, assign, and inspect true/false values
- Relational Operators for Comparisons — <, >, <=, >=, ==, ~= on scalars
- Row and Column Vector Concepts — create vectors, query size/length/numel, understand orientation
- Vector Construction Methods — literals, concatenation, colon notation, linspace
- Array Creation Utilities — zeros, ones, rand, eye, and similar constructors

Expect to think harder here:
- Logical Operators and Compound Tests — &, |, ~, xor; operator precedence and short-circuiting cause subtle bugs
- Elementwise Logical Operations on Arrays — apply relational/logical operators to vectors; must track shapes and elementwise semantics
- Integer (Subscript) Indexing of Arrays — select, reorder, and repeat elements with numeric index vectors; bounds errors and ordering mistakes are common
- Logical (Mask) Indexing of Arrays — extract or modify elements using logical masks; mask dimensions must match the array, and results can reshape unexpectedly
- Resizing and Assignment to Arrays — grow/shrink via indexed assignment and end; implicit array growth confuses beginners and quietly hurts performance

Conceptually demanding — start early:
- Numeric Comparison Nuances and Tolerances — compare floating-point arrays with abs(a-b) <= tol; direct == comparison of floats almost never does what you expect

Lecture 3 — Essential Skills

Vector arithmetic, operations, and basic visualization.

Build fluency with these:
- Elementwise Arithmetic on Vectors — +, -, .*, ./, .^; understand the . distinction from matrix operations
- Elementwise Relational Operations — <, >, ==, ~= on vectors to produce logical masks
- Aggregation and Cumulative Operations — sum, prod, cumsum, diff; global totals vs running results
- Numeric Utilities and Elementwise Transforms — round, floor, ceil, mod/rem applied to vectors
- Simple 2-D Plotting of 1-D Data — plot/stem, axis labels, titles, legends, saving figures programmatically

Expect to think harder here:
- Elementwise Logical Operations and Mask Combination — &, |, ~, xor on logical vectors; precedence and parentheses must be deliberate
- Logical (Mask) Indexing and Modification — select, assign, and remove elements using masks; mask shape and assignment effects are subtle
- Common Vector Functions and Shape-Aware Usage — sum, cumsum, diff, conv; output shapes differ by function — always check
- Random Number Generation and Reproducibility — rng seeds for repeatable experiments; easy to use, easy to misuse when writing tests

Conceptually demanding — start early:
- Complex-Valued Vectors and Basic Statistics — real, imag, abs, angle, mean/std; phase and magnitude interpretation is a new abstraction for most students

Lecture 4 — Essential Skills

2-D arrays (matrices): construction, indexing, and shape manipulation.

Build fluency with these:
- Creating and Inspecting 2-D Arrays — construct matrices, size/ndims/numel, understand column-major memory layout
- Element Access and Assignment with (row,col) Indexing — read/write A(i,j), handle out-of-range errors
- Row/Column Slicing with the Colon Operator — A(:,j), A(i,:), A(i,1:3); returned shapes are predictable
- Using end in Index Expressions — size-robust slices like A(end,:), A(1:end-1,end)

Expect to think harder here:
- Linear Indexing and Conversions — A(k), A(:), sub2ind/ind2sub; requires mapping between linear and subscript views in column-major order
- Concatenation of Matrices — [A,B] and [A;B]; dimension mismatches are the most common error here
- Multi-Element Indexing with Integer Index Vectors — A(rows,cols) including repeated/reordered indices; powerful but easy to get wrong
- Logical Masks for Selection and Assignment — A > v, A == v; extraction follows column-major order, which surprises most students
- Constructing Logical Index Expressions — combine relational and logical operators for conditional selection; parentheses required for correct precedence
- Reshaping and Linear View Operations — reshape, A(:), transpose (.' vs '); column-major ordering and conjugation are non-obvious
- Safe Shape Manipulation and Error Avoidance — use size/length to guide operations, check compatibility before indexing; defensive habits that prevent hard-to-debug errors

Lecture 5 — Essential Skills

Matrix manipulation: construction, reshaping, sorting, and reduction.

Build fluency with these:
- Inspecting Array Dimensions and Sizes — size, ndims, numel, length, isequal(size(A),[m n])
- Creating Common Matrices — zeros, ones, eye, diag, magic; default numeric class

Expect to think harder here:
- Concatenation and Block Construction — [A,B], [A;B]; dimension mismatches are frequent and the error message isn't always obvious
- Replication and Tiling — repmat and implicit expansion; easy to invoke, requires shape reasoning to use correctly
- Reshaping and Permuting Arrays — reshape, .^', permute; column-major ordering is the hidden trap
- Linear Indexing and find — A(:), A(k), find, sub2ind/ind2sub; mapping between linear and subscript views requires deliberate thought
- Logical Masking and Element Selection — relational expressions, isnan, isfinite, ismissing; NaN/missing handling is subtle and important
- Combining Masks and Logical Functions — &, |, ~ for conditional selection/cleansing; correct parenthesization matters
- Sorting and Reordering Data — sort/sortrows, apply permutation indices; consistently reordering rows and columns with the same permutation is error-prone
- Elementwise Operations and Implicit Expansion — .*, ./, .^; implicit expansion can silently produce unexpectedly large arrays
- Array Manipulation Utilities and Reduction Functions — flipud/fliplr, circshift, sum/mean/max/min with dimension arguments; forgetting the dimension argument is a common mistake

Lecture 6 — Essential Skills

Conditional logic: if/else, switch, and defensive branching.

Build fluency with these:
- If-Else Structure and Syntax — if … elseif … else … end blocks; mutually exclusive branches
- Relational Operators for Branching — <, >, <=, >=, ==, ~= in conditional expressions
- switch/case and Discrete Selection — clean selection among discrete alternatives including strings

Expect to think harder here:
- Multiway Branching with elseif — ordered multi-branch logic for graded decisions; wrong ordering silently misprioritizes checks
- Logical Operators in Conditions — &, |, ~, xor with parentheses; precedence errors produce conditions that look right but aren't
- Using any/all and Logical Masks for Branch Decisions — make branching decisions from elementwise comparisons; any vs all is easy to mix up
- Nested Conditionals and Structured Control Flow — readable, testable nested if/switch; deep nesting makes logic hard to verify or debug

Conceptually demanding — start early:
- Short-Circuit vs Elementwise Logical Operators — &&/|| vs &/|; using the wrong form in array contexts causes runtime errors, not just wrong answers
- Array Conditions: any/all and Truthiness Rules — arrays cannot be directly used as if conditions; the rules are non-obvious and the error message explains what, not why
- Input Validation and Defensive Branching — validate types/sizes/ranges, issue informative errors, provide safe defaults; designing genuinely robust validation takes more thought than the syntax suggests
- Edge-Case Handling: Empty, NaN, and Inf — isempty, isnan, isfinite; special values interact with logical and branching semantics in non-obvious ways

Lecture 7 — Essential Skills

Object-oriented programming: classes, properties, methods, and inheritance.

Expect to think harder here:
- Class Definition and Basic Constructor — classdef file, constructor that initializes properties from inputs; new file structure and syntax to learn together
- Declaring and Using Properties — define properties, Constant/Dependent attributes; basic declarations are easy; attributes and validation add complexity
- Instance and Static Methods — operate on object data vs callable without an object; knowing when to use static requires design judgment
- Simple Operator Overloading — overload plus, eq, etc. for intuitive object operations; mechanical once the pattern is clear, but consistent semantics need testing
- Arrays of Objects and Indexing — create, index, and concatenate object arrays; handle vs value behavior can produce surprising results

Conceptually demanding — start early:
- Access Control for Encapsulation — public, protected, private attributes; visibility design requires thinking about API boundaries up front
- Getters and Setters (Property Accessors) — set/get methods or Dependent properties; accessor semantics differ between value and handle classes in subtle ways
- Handle vs Value Class Semantics — reference vs copy behavior on assignment; this is the single most common source of OOP logic bugs for beginners
- Designing Stateful Classes (Mutating Behavior) — manage internal state with methods enforcing valid transitions; state invariants are easy to break and hard to diagnose
- Inheritance and Method Overriding — subclass, override methods, call superclass constructors; constructor chaining and correct overriding require careful design

Lecture 8 — Essential Skills

Numerical methods: fitting, integration, differentiation, and root finding.

Build fluency with these:
- Numerical Integration: Discrete Data — trapz/cumtrapz; account for nonuniform spacing and sampling resolution

Expect to think harder here:
- 1-D Interpolation Fundamentals — interp1 with 'linear'/'spline'/'pchip'; choosing the right method requires judgment about smoothness vs local accuracy
- Linear Least Squares Regression — A\b for overdetermined systems, residuals, test data evaluation; the solution is easy; deciding whether the model is actually good is not
- Numerical Integration: Functions — integral/integral2, tolerances, singularities; singularity handling and tolerance settings require care
- Root Finding for Scalar Equations — fzero and safeguarded Newton-Raphson; fzero is easy to call; implementing Newton robustly with convergence safeguards is not

Conceptually demanding — start early:
- Extrapolation Handling and Domain Guards — detect out-of-range queries, specify safe behavior; extrapolation silently produces wrong results — you won't always know it happened
- Polynomial Fitting and Limitations — polyfit/polyval, Runge phenomenon; high-degree polynomials can look great on training data and fail badly outside it
- Nonlinear Curve Fitting — nlinfit/lsqcurvefit, initial guesses, residual diagnostics; optimizer behavior depends heavily on the initial guess you provide
- Numerical Differentiation and Finite Differences — forward/central/backward differences, gradient; noise amplification makes this fundamentally different from analytic differentiation
- Solving Nonlinear Systems — fsolve, initial guesses, convergence diagnostics; multiple solutions may exist and the solver may find the wrong one
- Validation, Error Quantification, and Tolerance Selection — absolute/relative errors, tolerant comparisons; choosing a meaningful tolerance is a judgment call, not a formula you can look up

Lecture 9 — Essential Skills

Graphical user interfaces: building interactive apps and event-driven code.

Build fluency with these:
- Message and Alert Dialogs — msgbox/warndlg/errordlg/uialert; predictable behavior for simple user feedback

Expect to think harder here:
- Creating Figures and Basic UI Components — buttons, labels, sliders; Position/String/Value properties; coordinate-based positioning takes practice
- Collecting and Validating User Input — inputdlg or edit fields, type/range validation, clear feedback; validation logic inside UI code requires discipline
- Converting Scripts to Simple Interactive Apps — restructure procedural code for Live Scripts or simple GUIs; requires rethinking code flow, not just adding UI calls
- Dynamic UI Updates and Control Flow — enable/disable controls, update labels/plots dynamically; timing and consistency of updates is easy to get wrong
- Embedding and Updating Plots and Tables — axes/uitable inside GUIs, update from callbacks; correct data binding and redraw handling is required
- Input Validation and Immediate User Feedback — validation inside callbacks, UI gating to prevent invalid operations; responsive validation is important and moderately tricky
- App Designer Basics and Component APIs — ValueChangedFcn/callback wiring, design and code views; the component API takes time to internalize

Conceptually demanding — start early:
- Event-Driven Callbacks — callbacks execute on user interaction, not sequentially; switching from procedural to event-driven thinking is the biggest mental shift in this lecture
- State Management Across Callbacks — share state with guidata/app properties, not globals; incorrect state sharing produces bugs that appear to occur randomly
- Timers, Asynchronous Tasks, and Clean Cancellation — timer, background tasks, progress feedback, safe cleanup; concurrency is genuinely hard; cancellation logic is almost always forgotten

Lecture 10 — Essential Skills

Text and string processing in MATLAB.

Build fluency with these:
- Creating and Distinguishing Text Types — string scalars/arrays vs char vectors; isstring/ischar/isStringScalar
- Text Concatenation and Appending — +, append, strcat; know what type the result will be
- Measuring and Validating Text Objects — strlength, numel, scalar vs array checks
- Case, Whitespace, and Character Property Handling — lower/upper/strtrim, isstrprop/isletter/isspace

Expect to think harder here:
- Converting Between Text and Numeric/Container Types — string/char/cellstr, str2double/sscanf, num2str/sprintf; choosing the right conversion and handling errors takes care
- Text Comparison and Logical Masks — ==, strcmp/strcmpi, contains, startsWith, endsWith, matches; many APIs with subtle type differences between them
- Finding and Locating Substrings — strfind/contains/matches; different functions return different output types and shapes
- Extracting, Replacing, and Editing Text — extractBetween/Before/After, replace, strrep; many specialized functions — choosing the right one and preserving types is key

Conceptually demanding — start early:

  • Robust Formatting and Parsing of Text Data — sprintf/compose, sscanf/textscan; handling mixed or invalid input requires deliberate validation logic
    • Pattern Matching and Regular Expressions — we will not cover regular expressions in the course in any depth,yet they are used quite often in programming. regexp/regexpi; crafting correct patterns is a skill in itself, separate from MATLAB syntax

Lecture 11 — Essential Skills

Loops: while, for, convergence, and performance.

Build fluency with these:
- while Loop Syntax and Semantics — scalar logical condition, initialize and update loop variables
- for Loop Basics and Iteration Patterns — for i = 1:N, for x = v; know when to prefer vectorization instead
- break and continue Usage — early exit and iteration skipping; use judiciously to keep code readable

Expect to think harder here:
- Sentinel and Input-Driven Loops — loop until a sentinel value or user signal, validate termination conditions; interactive termination and robust input handling together is tricky
- Counting, Accumulation, and Running Statistics — sums/products, running mean/variance; numerically stable running statistics require careful initialization
- Building Arrays Inside Loops and Preallocation — preallocate before loops, avoid growing arrays; growing arrays in a loop is a performance trap that's very easy to fall into
- Nested Loops and 2-D Traversal Order — traverse matrices with nested loops; column-major order affects cache performance in ways that aren't obvious
- Looping with Arrays vs Elementwise Operations — recognize vectorization opportunities and refactor loops; knowing when and how to vectorize takes practice

Conceptually demanding — start early:
- Convergence Criteria and Tolerance-Based Iteration — absolute/relative tolerance stopping, max-iteration safeguards; wrong tolerances produce results that look converged but aren't
- Performance and Efficiency Techniques — minimize work inside loops, move invariant computations outside, use built-ins; requires understanding what MATLAB optimizes and what it doesn't
- Debugging, Testing, and Edge-Case Handling — empty/singleton/NaN inputs, breakpoints, infinite-loop prevention; defensive loop design requires thinking about failure modes up front, not after

Lecture 12 — Essential Skills

Structures: creating, indexing, and designing hierarchical data.

Build fluency with these:
- Creating Structures — struct and dot notation, valid field-name rules, isstruct
- Accessing and Assigning Fields — s.field, nested s.sub.field, guard against missing fields with isfield
- Common Structure Utilities — fieldnames, isfield, rmfield, orderfields

Expect to think harder here:
- Structure Arrays and Indexing — s(1), s(2).field, extracting fields across elements; understanding when you get a numeric array vs a cell array from field extraction is a common source of confusion
- Dynamic Field Names and Programmatic Creation — s.(fname), validate with isvarname; easy to write, easy to break with invalid or unexpected names
- Converting Between Containers — struct2cell, cell2struct, struct2table, table2struct; ordering and dimensional implications need attention
- Passing and Returning Structures in Functions — optional fields, isstruct/isfield validation, documenting expected fields; flexible struct API design requires care
- Efficient Preallocation and Bulk Assignment — s(n) = struct(...) before loops; growing struct arrays inside loops is a common performance mistake
- Logical Selection and Aggregation — [s.value] > thr, arrayfun/cellfun; type and shape surprises are frequent

Conceptually demanding — start early:
- Designing Nested/Hierarchical Data Layouts — model hierarchical data, traverse/flatten nested fields, know when tables or classes are a better fit; good design here requires thinking about the entire data lifecycle
- Robustness and Edge-Case Handling — empty structs, missing or heterogeneous fields, defaults, reliable comparison with isequal; handling missing or inconsistent fields defensively is harder than it looks

Lecture 14 — Essential Skills

Cell arrays, categorical arrays, and tables.

Build fluency with these:
- Cell Array Construction — {} and cell, preallocate for heterogeneous types/sizes
- Table Construction and Basic Manipulation — build from arrays/cells/variables, name columns, add/remove rows and columns

Expect to think harder here:
- Indexing Cells: Contents vs Containers — () for cell slices, {} for contents; () vs {} is the most common cell array mistake
- Dynamic and Programmatic Cell Access — C{idx}, comma-separated list unpacking; output shapes and compatibility require attention
- Cell-Array Utilities and Conversions — cellfun, cell2mat, num2cell, mat2cell; element types and sizes must be compatible or errors are cryptic
- Passing Cells To/From Functions — varargin/varargout, comma-separated lists; API design with variable numbers of inputs/outputs needs careful thought
- Categorical Arrays: Creation and Inspection — categorical, category levels, ordered vs unordered; ordered vs unordered semantics affect comparisons and grouping in non-obvious ways
- Extracting Data From Tables — dot notation, (), {}; each produces a different output type — use the right one for downstream operations

Conceptually demanding — start early:
- Managing Categories: Reorder, Merge, Missing — rename/add/remove/reorder categories, handle undefined values; subtle category management errors silently corrupt analysis
- Grouping, Aggregation, and Joins With Tables — groupsummary, rowfun, join/innerjoin/outerjoin; relational thinking and handling of keys and missing values are new skills for most students
- Import/Export Workflows and Choosing Containers — readtable/writetable/datastore, choosing table vs cell vs categorical; the right container depends on the whole analysis pipeline, not just the immediate step

Lecture 15 — Essential Skills

Functions in depth: scope, handles, recursion, and robust design.

Build fluency with these:
- Scripts with Local Functions — place local functions at the end of a script; scope is limited to that file
- Anonymous Functions for Compact Behavior — @ for short expressions and callbacks; variables are captured by value

Expect to think harder here:
- Functions with Subfunctions — primary + subfunctions in one file; each has its own separate workspace; subfunction workspaces are isolated from each other and from the caller
- Function Handles and Higher-Order Programming — create/pass/invoke handles, use in callbacks and generic algorithms; using handles in generalized APIs takes practice
- Persistent Variables for Retained State — persistent, initialize safely on first call, provide explicit reset patterns; correct initialization and reset are easy to get wrong
- Recursion and Termination Safeguards — base cases, recursion depth monitoring, input validation; robust recursion requires thinking about failure modes before coding
- Flexible Argument and Output Handling — nargin/nargout/varargin/varargout; clear optional-argument API design and validation requires thought

Conceptually demanding — start early:
- Nested Functions and Shared Workspace — share and mutate parent workspace variables without explicit argument passing; shared state produces non-obvious side effects that are hard to trace
- Workspaces and Variable Scope — base/script/function/nested workspaces, lifetime and visibility; scope bugs are among the hardest to diagnose because the symptom appears far from the cause
- Global Variables: Use and Alternatives — global shares state across functions; prefer argument passing, handles, or persistent; globals are easy to use and nearly impossible to reason about in large code — avoid unless necessary
- Defensive Function Design and Error Handling — validate inputs, assert/error/warning, comprehensive edge-case tests; writing robust, well-tested functions is a discipline that takes sustained practice

Lecture 16 — Essential Skills

Vectors and matrices: linear algebra foundations.

Build fluency with these:
- Row and Column Vectors, Orientation, and Sizes — construct, transpose, size/length/isempty
- Vector Arithmetic and Elementwise Operations — +, -, scalar scaling, .*, ./, .^; scalar broadcasting
- Vector Norms and Measures — norm(v,1), norm(v,2), norm(v,Inf) for magnitude and error comparison

Expect to think harder here:
- Inner (Dot) and Outer Products — dot, matrix multiplication for dot product, v * w.' for outer product; shapes and geometric interpretation (projections, rank-1 matrices) need attention
- Matrix Creation, Shape, and Indexing — zeros/ones/eye/diag, linear vs subscript indexing; indexing subtleties are a common source of errors
- Transpose vs Conjugate Transpose — .' vs '; trivial for real data; critical to get right for complex data
- Matrix Multiplication and Elementwise Matrix Ops — * vs .*; ensure dimension compatibility; the single most common matrix operation error in MATLAB
- Special Square Matrices and Basic Properties — eye/diag, symmetry tests, det/trace, singular vs nonsingular; numerical near-singularity adds nuance beyond the API
- Triangular Systems and Forward/Backward Substitution — identify lower/upper triangular, solve with backslash; conceptually clear, but correct indexing for manual substitution takes care

Conceptually demanding — start early:
- Elementary Row Operations and Elementary Matrices — represent row operations as matrix multiplications and observe effects on determinants; connecting operations to matrices requires linear-algebra insight that takes time to develop
- Solving Linear Systems with Backslash — A\b for square, overdetermined, and underdetermined systems; avoid explicit inverses; the call is simple; interpreting behavior for ill-conditioned, singular, or rectangular systems is not
