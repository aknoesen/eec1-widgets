Track A: Kemper Hall Tracker — Grading Rubric (Instructor Use Only)

Total: 100 points. All team members receive the same score unless a peer-reported imbalance triggers review.

Reference dataset (the "golden walk") is assumed to be available at the start of the grading window. Dataset contents: GPS trace (.mat), accelerometer trace (.mat), ground-truth annotation file listing entry time, each stair transition, each elevator start and stop, and exit time.

Grading script runs once per team. Each line is pass or fail.

Teammate evaluation form (required, separate from Cat 1). Each student independently submits a confidential teammate evaluation form (see Assessment Guide for the six statements). Submission of this form is a hard requirement but is not scored within the 25-point Cat 1 structure. It feeds into individual grade adjustments per the "Teammate Evaluation Handling" section below.

Category 2: Quantitative Performance Against Reference (30 points, Automated)

Each team's submitted code is run against the golden walk reference dataset. Team's app must expose, directly or through a simple test harness, a function analyzeReferenceWalk(referencePath) that returns a struct with the fields below. Teams that do not provide the test harness receive zero on items that require it; they can still earn points for items read directly from their ThingSpeak channel.

Category 5: Excellence, Polish, or Stretch (5 points, Grader Judgment)

Bonus category. Graders should use this to reward work that genuinely stands out.

  • GPS-anchored correction stretch challenge attempted and documented: up to 4 points
  • Exceptional polish, clarity, or engineering insight: up to 3 points
  • Innovative reach feature well beyond the suggested menu: up to 3 points

Maximum 5 points total. This category rewards effort above the baseline; it is not a default gift.

Teammate Evaluation Handling (Instructor Use)

The teammate evaluation form is a hard requirement (each team member submits one) but is not scored as part of the 100-point rubric. It is the instructor's primary input for deciding whether to adjust individual grades away from the team grade.

Default behavior. All three team members receive the same total score (sum of Cat 1 through Cat 5). No adjustments are made unless the teammate evaluation forms surface a clear contribution imbalance.

When to adjust.

  1. After all submitted forms are received, review each team's set of three.
  2. If all team members rate each other comparably across the six statements (no significant outliers), no adjustment.
  3. If a clear pattern emerges (for example, two teammates independently rate the third low on multiple contribution-related statements such as #1, #2, #4, or #5), the instructor follows up. Corroborating evidence (commit history, presence in the video, response to peer review patterns) is consulted before any adjustment.
  4. Adjustment magnitude is at the instructor's discretion based on severity and corroborating evidence. Document the rationale in the master spreadsheet's Notes column.
  5. Symmetric upward adjustment is possible for an individual demonstrably contributing significantly above teammates, but the threshold is higher and corroborating evidence is required.
  6. If a team member fails to submit their teammate evaluation form, the instructor follows up with that student. Default team grading proceeds unless the missing submission obstructs review of a flagged imbalance.

Confidentiality. Forms are reviewed only by the instructor. Teammates do not see each other's ratings.

Grader Workflow per Team

  1. Open grading_results.csv for this team (populated by the automated scripts, Categories 1 and 2).
  2. Read the two peer reviews for this team.
  3. Watch the video at 1.25x to 1.5x, scoring Category 3.
  4. Read the PDF, scoring Category 4.
  5. Award Category 5 points if warranted.
  6. Enter all scores in the master spreadsheet.
  7. Compare team's self-assessment to your total. If divergence is greater than 10 points, flag for discussion.

Target time per team: 15 minutes active grading (Categories 3 through 5) plus 2 minutes data entry. Automated categories (1 and 2) require no per-team grader time.

Reference Dataset Specification (For Instructor Use During Dataset Preparation)

The golden walk is recorded once, before the project assessment window opens. Contents:

  • golden_gps.mat: poslog output from an uninterrupted walk of the full Kemper Evaluation Route.
  • golden_accel.mat: accellog output synchronized to the GPS log.
  • golden_truth.json: timestamped annotations:
  • walk_start_time: wall clock at Start Activity
  • entry_time: when the walker crossed the building entrance
  • stair_events: array of (start, end, direction) tuples for each stair climb or descent
  • elevator_events: array of (start, end, direction, floors_traveled) tuples
  • floor_checkpoint_times: array of (time, floor_number) pairs at announced landmarks
  • exit_time: when the walker exited the building
  • walk_end_time: wall clock at Stop Activity

The grading script harness loads these three files, runs the team's analyzeReferenceWalk function, and produces the auto-grade row for Category 2.

Notes for Calibration Meeting

Before TAs begin grading, review three videos together (one expected-strong, one expected-average, one expected-weak) and align on Category 3 and Category 4 scoring. Particular attention:

  • What counts as "demonstrably working" in Category 3? Must see end-to-end flow, not individual components tested in isolation.
  • What counts as "consistent with Category 2 numbers"? Allow ±20 percent tolerance for stated numbers vs. auto-graded numbers, more if student explains the discrepancy.
  • Category 5 should be used sparingly. Default is 0. Award only for work that is clearly above typical submissions.
