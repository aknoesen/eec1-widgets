Total: 100 points. All team members receive the same score unless a peer-reported imbalance triggers review.

Reference dataset (gesture test streams and network test harness) is assumed to be available at the start of the grading window.

Category 2: Quantitative Performance Against Reference (30 points, Automated)

Each team's code is run against two reference scenarios. Team's code must expose a function testGestureDetector(referenceStream) that accepts an Nx3 accelerometer matrix and returns a boolean array of the same length indicating when the detector fires. Teams that do not provide the test harness receive zero on items requiring it.

Category 5: Excellence, Polish, or Stretch (5 points, Grader Judgment)

Bonus category for work that stands out above typical submissions.

  • Exceptional architectural clarity: code separation is so clean that reach features truly drop in without modifying other classes: up to 3 points
  • Innovative game choice or rule design that increases project difficulty meaningfully: up to 3 points
  • Polish: sound, animations, UI design that reflects real care: up to 3 points
  • Notable use of the ThingSpeak platform: multi-channel design, leaderboard that updates across sessions, spectator view that actually works: up to 3 points

Maximum 5 points total.

Teammate Evaluation Handling (Instructor Use)

The teammate evaluation form is a hard requirement (each team member submits one) but is not scored as part of the 100-point rubric. It is the instructor's primary input for deciding whether to adjust individual grades away from the team grade.

Default behavior. All three team members receive the same total score (sum of Cat 1 through Cat 5). No adjustments are made unless the teammate evaluation forms surface a clear contribution imbalance.

When to adjust.

  1. After all submitted forms are received, review each team's set of three.
  2. If all team members rate each other comparably across the six statements (no significant outliers), no adjustment.
  3. If a clear pattern emerges (for example, two teammates independently rate the third low on multiple contribution-related statements such as #1, #2, #4, or #5), the instructor follows up. Corroborating evidence (commit history, presence in the video, response to peer review patterns) is consulted before any adjustment.
  4. Adjustment magnitude is at the instructor's discretion based on severity and corroborating evidence. Document the rationale in the master spreadsheet's Notes column.
  5. Symmetric upward adjustment is possible for an individual demonstrably contributing significantly above teammates, but the threshold is higher and corroborating evidence is required.
  6. If a team member fails to submit their teammate evaluation form, the instructor follows up with that student. Default team grading proceeds unless the missing submission obstructs review of a flagged imbalance.

Confidentiality. Forms are reviewed only by the instructor. Teammates do not see each other's ratings.

Grader Workflow per Team

  1. Open grading_results.csv for this team (Categories 1 and 2 populated by scripts).
  2. Read the two peer reviews for this team.
  3. Watch the video at 1.25x to 1.5x, scoring Category 3.
  4. Read the memo, scoring Category 4.
  5. Award Category 5 points if warranted.
  6. Enter all scores in the master spreadsheet.
  7. Compare team's self-assessment to your total. If divergence is greater than 10 points, flag for discussion.

Target time per team: 15 minutes active grading (Categories 3 through 5) plus 2 minutes data entry.

Reference Dataset Specification (For Instructor Use During Dataset Preparation)

The gesture test streams are recorded once, before the project assessment window opens. Contents:

  • gesture_stream_1.mat: 90 seconds of accelerometer data with 12 deliberate shake events at annotated timestamps, plus quiet periods.
  • gesture_stream_2.mat: 60 seconds with mixed motion (typing nearby, walking, phone handling without intentional shakes) containing zero intentional shake events. Used to measure false-positive rate.
  • gesture_truth.json: timestamps of all intentional shake events in the first stream.

Network test harness:

  • A MATLAB script that, given a team's NetworkMessenger class and their ThingSpeak channel URL, writes two tag moves and verifies the team's reader retrieves them in the correct order.

The grading script harness produces the auto-grade row for Category 2 from these resources.

Notes for Calibration Meeting

Before TAs begin grading, review three videos together (one expected-strong, one expected-average, one expected-weak) and align on Category 3 and Category 4 scoring. Particular attention:

  • What counts as "automatic turn passing"? The other player's screen must update without any user input on either machine. Debug prints or console output do not count.
  • What counts as "four separated concerns" in Category 3? Four distinct class files with a clear single responsibility each. A single class that contains both network code and game state logic fails this item regardless of naming.
  • Category 5 should be used sparingly. Default is 0. Award only for work clearly above typical submissions.

Common Failure Modes to Watch For

  • "Demo only on one computer." Some teams will pass a game state channel locally (two MATLAB sessions on the same machine) and stage a video that suggests two machines. Check the architecture walkthrough for network code that actually uses ThingSpeak; check the ThingSpeak channel history for round-trip writes from two distinct clients.
  • Gesture-detector false-positive exploitation. A team sets the shake threshold so low that the game advances on any motion, demonstrating reliable triggering in the video but with a high false-positive rate on the reference stream. Category 2 penalty handles this.
  • Mixed-concern class that does everything. A single class named "GameApp" that contains sensor reading, threshold logic, game rules, and ThingSpeak calls. Category 1 and Category 3 both penalize; total impact is meaningful.
  • Stub reach feature. A reach feature described in the memo but not actually working. Video review catches this; Category 3 and Category 4 both lose points.
