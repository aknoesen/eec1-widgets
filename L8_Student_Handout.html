Key Concepts (keep this as a study reference)

  • cumtrapz(t, y) returns an array — the running integral at every time step. Use when you need to plot or further process the integral.
  • trapz(t, y) returns a scalar — the total area. Use when you need one number.
  • The last element of cumtrapz(t,y) equals trapz(t,y) — both give the total, but only cumtrapz gives the trajectory.
  • Numerical integration is approximate. Error is smaller with more samples (shorter Δt).
  • For dead reckoning: integrate acceleration → velocity, integrate velocity → displacement. Two cumtrapz calls.
  • diff(y) returns an array of length N−1. Always account for this when plotting.

Key Concepts (keep this as a study reference)

  • A constant bias in acceleration produces quadratically growing drift in displacement (not linear) — because the error accumulates through two integrations.
  • Mean subtraction on the acceleration reduces DC offset but does not fully remove accumulated integration error if any residual remains.
  • In practice, drift is managed by: (a) careful bias estimation from a stationary window, (b) periodic resets using external reference (GPS, landmarks), or (c) sensor fusion.
  • polyfit on a drifting signal can characterize the trend — but this is a post-processing correction, not a fix to the underlying physics.
  • The MP4 gravity offset must be measured before motion begins — using mean(Az) during a stationary window.
  • Dead reckoning is most accurate over short time windows. The longer you integrate, the larger the accumulated error.
