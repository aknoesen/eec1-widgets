Before you write that formula in code, you need to understand what it represents and why it is useful.

This is a before concept — you need to understand it before the lab session, not after. Students who arrive without this understanding will implement the formula correctly but not know what it means or when it applies.

Lab Step 1: Data Collection

Background: File I/O

In Lab 1 you collected data interactively and examined it in the command window. In this lab you will save your data to a file so it can be reloaded later — by your own analysis script, and by a partner running their analysis on your data.

File I/O (input/output) refers to writing data from a program to a permanent file on disk, and reading it back in a new session or program. MATLAB's save and load functions handle this simply — consult the documentation for the full syntax, including options for saving individual variables and loading into specific workspaces:

% Writing data to a file
save('filename.mat', 'variableName')

% Reading data back in
load('filename.mat')

A .mat file stores MATLAB variables in a format that preserves their exact values and structure. Unlike a CSV file, it handles arrays, timestamps, and mixed data types cleanly.

Lab Step 2: Data Analysis Script

Create a new script called dataAnalysis.m in the same MiniProject2 folder.

This script will load your saved data, compute several derived quantities, produce an annotated plot, and save the result. Build it section by section — run it after each addition to catch errors early.

Section 1: Load the Data

load('sensorData.mat')

This restores the sensorData variable from the file you saved in Lab Step 1. Use array indexing to extract each column into named variables for clarity — for example:

Ax = sensorData(:, 1);   % X acceleration
Ay = sensorData(:, 2);   % Y acceleration
Az = sensorData(:, 3);   % Z acceleration
t  = sensorData(:, 4);   % timestamps

Convert timestamps to elapsed time starting from zero:

t = t - t(1);

Section 2: Compute the Resultant Acceleration

Before writing this line, recall your Pre-Lab AI Exploration: the resultant gives the total magnitude of the acceleration vector, independent of direction.

A_resultant = sqrt(Ax.^2 + Ay.^2 + Az.^2);

Note the .^2 syntax — this squares each element of the array individually (element-wise), which is what you want here.

Section 3: Create a Four-Panel Plot

Your plot must have four subplots arranged vertically in a single figure:
- Panel 1: |Ax| vs. time
- Panel 2: |Ay| vs. time
- Panel 3: |Az| vs. time
- Panel 4: Resultant acceleration vs. time

Use the subplot function to create multiple panels in one figure — the documentation's "Quadrant of Subplots" example maps directly onto this task. Each panel must have:
- A descriptive title
- Labeled axes with units (time in seconds, acceleration in m/s²)

Section 4: Peak Detection

For each of the four signals, find the time at which the peak magnitude occurs. Store all four elapsed times in a 1×4 array called peakTimes:

peakTimes = [peakTimeX, peakTimeY, peakTimeZ, peakTimeResultant];

where items 1–3 are the elapsed times to peak magnitude in X, Y, and Z respectively, and item 4 is the elapsed time to peak resultant acceleration.

Hints:
- Use max to find the maximum value and its index in an array — pay attention to the two-output form [val, idx] = max(...) which returns both the value and its position
- Use the index to look up the corresponding time value from your timestamp array
- Elapsed time = timestamp at peak − first timestamp (already handled if you zeroed t in Section 1)

Write down your four peakTimes values in your notes document right now. You will enter them in your post-lab submission.

Section 5: Annotate the Plot

For each of the four panels, add:

  1. A vertical red line at the time of peak acceleration magnitude. Use xline to add a vertical reference line to the axes, or plot a vertical line manually.
  2. A text label adjacent to the red line showing the peak magnitude value. Use the text function, offset slightly from the line so it does not overlap.

A small amount of vertical padding above each subplot helps prevent annotations from being clipped — consider using ylim to add headroom above the maximum value.

Section 6: Save the Figure

Save the completed annotated figure as a PNG file named accelerationPeaks.png. Use exportgraphics to save the figure with a specified resolution — the documentation covers the full range of output format and resolution options:

exportgraphics(gcf, 'accelerationPeaks.png', 'Resolution', 150)

Before saving, use sgtitle to add your name as a title spanning all four panels:

sgtitle('Acceleration Analysis — Your Name Here')
